# Project Overview Update Rule

## When to Update

Update `.kiro/docs/PROJECT_OVERVIEW.md` whenever:

1. **New feature completed** - Add to "Completed Features" table
2. **Version bump** - Update "Current Version" and "Version History"
3. **New component added** - Update "Project Structure" section
4. **New API endpoint** - Update "API Endpoints" section
5. **New template/theme** - Update "Templates & Themes" section
6. **Architecture change** - Update "Architecture" section

## Update Checklist

- [ ] Update "Last Updated" date at top
- [ ] Update version number if changed
- [ ] Add new features to completed table
- [ ] Update file counts if significant changes
- [ ] Keep structure diagram current
- [ ] Sync with CHANGELOG.md

## File Location

`.kiro/docs/PROJECT_OVERVIEW.md`
